# GEMINI.md - Paulo Chaves (opchaves) Personal Website & Blog

## Project Overview
This is a modern, full-stack personal website and blog engine built by Paulo Chaves. It leverages React Router v7 (formerly Remix) for server-side rendering and a seamless developer experience, deployed on Cloudflare Pages/Workers.

### Core Tech Stack
- **Framework:** [React Router v7](https://reactrouter.com/) (SSR, HMR, Data Loading)
- **Language:** TypeScript
- **Database:** Cloudflare D1 (SQLite-based) with [Drizzle ORM](https://orm.drizzle.team/)
- **Authentication:** [Better Auth](https://www.better-auth.com/)
- **Styling:** Tailwind CSS 4.0
- **Deployment:** [Cloudflare Pages](https://pages.cloudflare.com/) via Wrangler
- **Testing:** Playwright for E2E testing
- **Content:** Markdown-based blog posts with `react-markdown` and `uiw/react-md-editor`

## Directory Structure
- `app/`: Main application source code.
  - `routes/`: React Router route definitions (Home, Blog, Auth, Dashboard).
  - `components/`: Reusable UI components.
  - `welcome/`: Welcome page assets and components.
- `database/`: Database schema and migrations.
  - `schema.ts`: Core application schema (posts, comments, tags).
  - `auth-schema.ts`: Authentication-related tables (users, sessions).
  - `migrations/`: SQL migration files generated by Drizzle.
- `lib/`: Shared utilities and server-side logic.
  - `auth.server.ts`: Server-side authentication configuration.
  - `env.ts`: Environment variable validation and access.
- `workers/`: Entry point for the Cloudflare Worker/Pages Function.
- `tests/`: End-to-end tests using Playwright.
- `public/`: Static assets (favicon, robots.txt, PDF resume).

## Building and Running

### Development
1. **Install Dependencies:**
   ```bash
   npm install
   ```
2. **Environment Setup:**
   Copy `.env.example` to `.env` (if applicable).
3. **Database Setup (Local):**
   ```bash
   npm run auth:db:generate
   npm run db:generate
   npm run db:migrate
   ```
4. **Start Dev Server:**
   ```bash
   npm run dev
   ```

### Testing
- **E2E Tests:**
  ```bash
   npm run test:e2e
  ```

### Production & Deployment
1. **Build Project:**
   ```bash
   npm run build
   ```
2. **Production Migrations:**
   ```bash
   npm run db:migrate-production
   ```
3. **Deploy to Cloudflare:**
   ```bash
   npm run deploy
   ```

## Development Conventions
- **Routing:** Uses React Router's file-based routing but configured in `app/routes.ts`.
- **Validation:** Uses `zod` for schema validation (forms, env vars).
- **Forms:** Uses `react-hook-form` with `@hookform/resolvers/zod`.
- **Database:** Always use `drizzle-kit` to generate and apply migrations.
- **Styling:** Tailwind CSS 4.0 is used; prefer utility classes over custom CSS where possible.
- **Authentication:** Rely on `better-auth` primitives. Auth routes are handled under `/api/auth/*`.

## Key Files
- `wrangler.jsonc`: Cloudflare configuration including D1 database bindings.
- `drizzle.config.ts`: Drizzle ORM configuration for migrations and schema management.
- `react-router.config.ts`: React Router framework configuration.
- `vite.config.ts`: Vite build and plugin configuration.
- `worker-configuration.d.ts`: TypeScript types for Cloudflare environment bindings.
